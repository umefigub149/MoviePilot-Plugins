# 联动115追更

这个插件做一件简单的事：

```text
115追更插件 或 STRM助手 分享转存成功
→ 等几秒
→ 把你填好的目录交给 MoviePilot 整理
```

## 每个功能是什么意思

### 启用插件

打开后插件才会工作。关闭后不会自动检查，也不会自动整理。

### 演练模式（只记录日志，不真正入队）

就是“只试跑，不真干”。

打开后：

- 会检查有没有新的转存记录。
- 会告诉你本来要整理哪个目录。
- 不会真的交给 MoviePilot 整理。
- 不会移动文件、刮削、入库。

第一次使用建议先打开它，确认目录没填错，再关闭。

### 第一次启用时跳过以前的记录

打开后，插件第一次启用时不会处理以前的老记录，只从当前最新记录之后开始。

日志里的 `记录ID=4221` 不是 4221 条记录，只是数据库给这条记录的编号。你可以理解成“书签位置”：插件记住已经看到哪里了，下次只看后面的新记录。

### 追更转存后自动整理

115追更插件转存成功后，本插件会把你配置的“追更成功后要整理的目录”交给 MoviePilot 整理。

### STRM助手分享转存后自动整理

STRM助手分享转存成功后，本插件会把你配置的“分享转存成功后要整理的目录”交给 MoviePilot 整理。

### 等待几秒再整理

转存刚完成时，网盘或 CloudDrive2 可能还没刷新出来。等几十秒再整理，更稳。

### 目标存储

告诉 MoviePilot 这些目录属于哪个存储。

常见选择：

- `CloudDrive储存`：使用 CloudDrive2 时通常选这个。
- `u115`：使用 MP 的 u115 存储时选这个。
- `115网盘Plus`：使用 115网盘Plus 时选这个。
- `local`：目录是容器本地路径时选这个。

### 要整理的目录

一行一个目录。插件不会猜具体新文件夹，只会把这些固定目录交给 MoviePilot，让 MoviePilot 自己递归扫描。

### 只允许整理这些路径下面的目录

安全保护。只有这里列出的路径下面的目录才允许整理。不要填 `/`。

### 短时间内防重复整理秒数

避免同一个目录短时间内重复交给 MoviePilot。一般默认 300 秒就可以。

## 推荐配置

```text
目标存储：CloudDrive储存

追更成功后要整理的目录：
/网盘整理/网盘待整理目录/Movie
/网盘整理/网盘待整理目录/TV

分享转存成功后要整理的目录：
/最近接收
/网盘整理/分享转存目录

只允许整理这些路径下面的目录：
/网盘整理
/最近接收
/我的接收
```

## 和 STRM助手网盘管理的区别

STRM助手的网盘管理是大功能：管理 115 文件、STRM、缓存、移动、删除、生命周期等。

本插件不是第二个 STRM助手。本插件只负责：

```text
发现成功转存 → 稍等 → 通知 MP 整理固定目录
```

本插件不做：

- 不扫描 115 网盘目录。
- 不猜这次新增的是哪个子目录。
- 不做目录对比。
- 不读取 Cookie。
- 不删除、移动、重命名 115 文件。
- 不接管 MoviePilot 的整理流程。

## 记录ID不是记录条数

日志里的 `最新记录ID=3457` 表示 MoviePilot 数据库里那条记录的编号，不是“3457 条记录”。

插件真正关心的是：上次看到哪个编号，这次有没有编号更大的新记录。
